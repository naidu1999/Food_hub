# 🍽️ FoodHub Order Data Analysis Project

## 📖 Project Description

This project focuses on analyzing food delivery order data collected by **FoodHub**, an online food aggregator platform operating in New York. FoodHub connects customers with multiple restaurants through a single mobile application and manages the end-to-end food delivery process.

The main goal of this project is to **analyze customer orders, restaurant demand, delivery performance, and ratings** using data analysis techniques. The insights generated from this analysis can help FoodHub improve customer experience, operational efficiency, and business decision-making.

---

## 🎯 Objective of the Project

The objectives of this project are:

- To understand the structure and quality of the dataset
- To analyze customer ordering behavior
- To identify popular restaurants and cuisines
- To study food preparation time and delivery time
- To understand how ratings are influenced by different factors
- To provide data-driven business recommendations

---

## 🗂 Dataset Overview

The dataset contains information related to food orders placed by registered customers on the FoodHub platform.

### Key Features in the Dataset

- **Order Information** – Order ID, customer ID
- **Restaurant Details** – Restaurant name and cuisine type
- **Order Value** – Cost of the order
- **Time Factors** – Food preparation time and delivery time
- **Customer Feedback** – Ratings given by customers
- **Order Timing** – Weekday or weekend orders

Each row in the dataset represents **one completed food order**.

## 🔍 Work Done in This Project

The project is divided into structured phases to ensure clear and logical analysis.

---

### 1️⃣ Data Understanding & Preparation

- Loaded the dataset and inspected its structure
- Checked dataset shape and column data types
- Generated statistical summaries for numerical features
- Identified and handled missing values where required

This step helped in building a strong foundation before performing any analysis.

---

### 2️⃣ Univariate Data Analysis

In this phase, each variable was analyzed individually to understand its distribution and behavior.

Key analysis performed:
- Distribution of order costs
- Distribution of customer ratings
- Analysis of food preparation time
- Analysis of delivery time
- Frequency of different cuisines
- Order count for weekdays vs weekends

Visualizations such as **histograms, bar plots, and count plots** were used along with clear observations.

---

### 3️⃣ Multivariate Data Analysis

This phase focused on understanding relationships between multiple variables.

Key relationships explored:
- Order cost vs customer rating
- Cuisine type vs delivery time
- Restaurant popularity vs order volume
- Day of the week vs delivery and preparation time
- Rating patterns across different restaurants and cuisines

Box plots, grouped bar charts, and comparative visualizations were used to support insights.

---

### 4️⃣ Data Visualization

- Appropriate visualizations were chosen based on the type of data
- Graphs were clearly labeled and easy to interpret
- Observations were written alongside each plot to explain insights

Visualization played a key role in identifying trends and patterns.

---

## 📊 Key Insights from the Analysis

- Certain cuisines such as American, Japanese, Italian, and Chinese are highly popular
- Some restaurants consistently receive higher ratings and more orders
- Weekend orders are generally higher than weekday orders
- Longer delivery times can negatively impact customer ratings
- Efficient food preparation and timely delivery contribute to better customer satisfaction

---

## 💡 Recommendations

Based on the analysis, the following recommendations are suggested:

- Promote high-demand cuisines to attract more customers
- Collaborate closely with top-rated restaurants
- Increase delivery workforce during weekends
- Optimize delivery routes to reduce delivery time
- Encourage customers to provide ratings for better feedback analysis
